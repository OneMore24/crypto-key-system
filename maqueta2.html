Sistema de Generación y Distribución de Claves Criptográficas

Maqueta 2 — el mismo sistema, mostrado como un proceso guiado paso a paso (más fácil de seguir el flujo completo).

1

Registrar usuarios

Antes de poder comunicarse de forma segura, cada persona debe registrarse en el sistema. Al hacerlo, se le genera automáticamente un par de claves (pública y privada).

creado Usuario "ana" registrado → par de claves RSA-2048 generado y guardado.
↓
2

Elegir quién quiere hablar con quién

Para iniciar una conversación segura, se elige un emisor y un receptor. El sistema comprobará que ambos estén registrados y tengan su clave pública disponible.

de para
↓
3

El sistema genera una clave de sesión

El servidor crea una clave simétrica (AES) nueva y temporal, que será la que ana y luis usen para cifrar sus mensajes entre ellos.

generada Clave de sesión AES-256: 6f1e9c2a4b7d8e3f...c91a (temporal, solo para esta conversación)
↓
4

El sistema distribuye la clave de forma segura

La clave de sesión se cifra por separado con la clave pública de ana y la de luis, así solo ellos —con su clave privada— pueden abrirla. Esto evita que alguien que intercepte el mensaje pueda leer la clave.

enviado a ana paquete cifrado con su clave pública: 3a9f...e21c
enviado a luis paquete cifrado con su clave pública: 8b02...77af
↓
5

Listo: ya pueden comunicarse cifrado

Cada uno descifra el paquete con su propia clave privada, obtiene la clave de sesión y a partir de ahí cifra/descifra sus mensajes con ella. Todo el proceso queda registrado para poder revisarlo después.

log 10:08 — Clave de sesión generada y entregada a ana y luis. Comunicación segura activa.
