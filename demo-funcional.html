Sistema de Generación y Distribución de Claves Criptográficas

Demo funcional — usa criptografía real del navegador (Web Crypto API: RSA-OAEP + AES-GCM). Todo ocurre en tu propio equipo, nada se envía a ningún servidor.

1

Registrar usuarios (genera su par de claves RSA-2048)

Cada usuario necesita un par de claves: una pública (se puede compartir) y una privada (secreta). El sistema las genera al registrarse.

El sistema crea una clave simétrica nueva, temporal, solo para esta conversación. (La mostramos aquí únicamente con fines didácticos — en un sistema real nunca se expone así).

4

Distribución cifrada (cifrado híbrido RSA + AES)

La clave de sesión se cifra por separado con la clave pública RSA de cada participante. Solo quien tenga la clave privada correspondiente podrá recuperarla.

— Aún no se ha distribuido nada —

5

Recibir, descifrar y probar la comunicación

Cada participante usa su clave privada para recuperar la clave de sesión, y luego ambos pueden cifrar/descifrar mensajes reales con ella (AES-GCM).
